# 项目提示词：Web 导航站 · 快捷方式管理器

## 项目概述
本项目是一个**现代化个人/团队快捷导航站**，类似浏览器起始页，用于高效管理常用链接。  
核心功能包括快捷方式的**增删改查、拖拽排序、分类收纳、快速搜索**，所有数据存储在本地，无需后端服务。  
要求使用最新技术栈实现，代码结构清晰，交互体验流畅，符合现代前端工程规范。

---

## 技术栈要求

| 模块 | 技术选型 | 说明 |
|------|--------|------|
| 核心框架 | **React 19** | 使用最新稳定版本，充分利用新特性 |
| 构建工具 | **Vite 7** | 极速启动与热更新，配置简洁 |
| 状态管理 | **Zustand** | 轻量级状态库，配合 `persist` 中间件实现本地持久化 |
| 拖拽交互 | **dnd-kit** | 实现快捷方式卡片的拖拽排序，支持鼠标与触摸 |
| 样式方案 | **原生 CSS** | 使用 CSS 变量 + Flex/Grid 布局，避免 CSS-in-JS 或大型框架 |
| 图标库 | **React Icons** | 集成多种图标集，便于用户选择图标 |
| 包管理 | **NPM** | 官方客户端，依赖管理清晰 |

## 核心功能需求

### 1. 快捷方式展示与管理
- 以卡片网格或列表形式展示快捷方式（包含图标、名称、URL）。
- **添加快捷方式**：用户可输入名称、URL，并从 React Icons 中选取图标，或自动获取网站 favicon 作为备选。
- **编辑/删除**：支持修改已添加快捷方式的信息，或将其移除。

### 2. 拖拽排序
- 所有快捷方式（或同一分组内）支持**拖拽重新排序**。
- 使用 `dnd-kit` 实现，拖拽过程中卡片有视觉反馈（阴影、位移、透明度变化），释放后顺序持久化保存。

### 3. 分类/分组
- 默认提供“常用”“工作”“娱乐”“学习”等预设分组，用户可**自定义分组名称**，并支持新增/删除分组。
- 分组标题可**折叠/展开**，节省屏幕空间。
- 快捷方式可自由分配到不同分组，支持跨组拖拽（可选加分项）。

### 4. 快速搜索/过滤
- 顶置搜索输入框，**实时过滤**当前视图中显示的快捷方式（按名称、URL 模糊匹配）。

### 5. 主题定制
- 基于 **CSS 变量**实现**浅色/深色模式**切换（至少提供手动切换开关）。
- 允许用户自定义主色调、背景模糊程度、卡片圆角等（作为进阶加分项）。

### 6. 数据持久化
- 使用 Zustand 的 `persist` 中间件，自动将状态（快捷方式列表、分组、排序、主题偏好）同步至 `localStorage`。
- 页面刷新或关闭后重新打开，数据完整恢复。

### 7. 响应式布局
- 适配**移动端、平板、桌面**三种尺寸，在小屏幕下拖拽仍可流畅操作（dnd-kit 传感器自动适配触摸事件）。

## 开发规范与交付标准

### 代码结构
- 项目根目录结构清晰，建议如下：
```
src/
  components/     # UI 组件（每个组件独立文件夹，含 JSX 和 CSS）
  stores/         # Zustand 状态存储定义
  utils/          # 辅助函数（如 favicon 获取、持久化配置）
  styles/         # 全局样式、CSS 变量定义
  App.jsx
  main.jsx
```
- 组件采用**函数组件 + Hooks**，避免类组件。
- CSS 文件与组件对应，使用 **BEM 或类似命名空间**，避免样式冲突。

### 语言与文本
- 所有**用户可见的界面文本**（按钮标签、提示信息、分组名称等）必须使用**简体中文**。
- 代码注释可使用中文或英文，但应在整个项目中保持统一。

### 构建与运行
- 提供完整的 `package.json`，包含所有依赖及以下 NPM 脚本：
  - `dev`：启动开发服务器
  - `build`：生产构建
  - `preview`：预览构建产物
- 确保 `npm run dev` 可直接运行项目，无额外配置步骤。

### 文档
- 根目录必须包含 `README.md`，内容包括：
  - 项目简介与截图（可选）
  - 技术栈清单
  - 功能特性
  - 本地启动与构建命令
  - 任何特殊的配置说明

### 交付物
1. **完整可运行的源代码**（建议提供压缩包或 GitHub 仓库链接）。
2. **项目实现步骤/思路说明**（可作为独立文档或在 README 中概述）。
3. **演示录屏或关键截图**（非强制，但推荐，便于直观评估）。

## 注意事项
- **功能完备性优先**：先确保所有核心功能稳定运行，再追求细节优化。
- **性能与体验**：拖拽交互需流畅无卡顿，状态更新应避免不必要的重渲染。
- **代码可维护性**：保持组件单一职责，避免巨型组件，逻辑复用使用自定义 Hooks。
